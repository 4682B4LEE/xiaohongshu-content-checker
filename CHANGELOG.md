# Changelog

所有 notable changes 将记录在此文件。

格式基于 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.0.0/)。

## [1.3.3] - 2026-06-12

### Added
- **首次使用欢迎语**：新用户触发时展示 Skill 能力、可用模式和示例指令
- **语义风险识别模块（双引擎检测）**：
  - 识别阴阳怪气/讽刺、软性导流、文化敏感、隐性承诺、情绪操控、虚假对比
  - 语义风险独立标注"基于语义分析"，不纳入 risk-rubric 计分体系
  - 与关键词引擎并行运行，互补验证
- **Golden Test 标准评测样例**（references/examples/golden-cases.md）：
  - 覆盖 /fast、/full、/batch、/video 四种模式
  - 包含反向诱导、空输入、非小红书文本等边界场景
  - 提供预期输出结构和验证要点，支持 Skill 测试和质量保证

### Changed
- **README 移至仓库根目录**：作为仓库级文档，与 Skill 代码分离
- **修正过度确定性措辞**：
  - "告别限流，发前必查" → "发前检查，降低风险"
  - "3 秒出报告" → "快速生成检查报告"
  - "统统逃不过" → "尽量识别常见变体"
  - "必须修改后发布" → "建议修改后再发布"
  - "完全合规可发布" → "较为合规可发布"
  - 所有 reference 文件统一使用建议性表述
- **批量检测输入分隔协议**：支持 `---` 分隔符、编号标记、JSON/YAML 列表三种方式
- **视频模式明确为叠加模式**：在 fast/full 基础输出上追加视频专项结果，不重复输出
- **移除 reference 文件内版本历史**：统一放 CHANGELOG.md，减少上下文占用
- **统一评分体系**：
  - risk-rubric.md 风险等级表扩展为四列：命中项分值、总分、等级、发布建议
  - 消除 demo 文件中 "9/10"、"必须修改后发布" 等与规则不一致的表达
  - mode-fast.md、mode-full.md 总体结论统一为四列格式
  - 所有文件版本号和日期更新至 1.3.2 / 2026-06-11
- **清理 Skill 包内杂质**：
  - 将 README.md、CHANGELOG.md、SKILL_LOGIC.md 移至仓库根目录
  - Skill 包内仅保留运行必要文件：SKILL.md + references/ 目录
  - 符合 Skill Creator 规范，减少非运行文件干扰
- **替换库合规性优化**：
  - 移除"主页置顶"等可能被理解为灰色导流的替换建议
  - 统一使用小红书官方认可的合规路径：小红书店铺、商品卡、蒲公英、官方邮箱、站内群聊
  - 明确标注替换方案均为官方认可的合规路径

## [1.3.1] - 2026-05-28

### Changed
- **唯一评分来源**：`risk-rubric.md` 设为 Skill 内唯一评分规则文件，其他文件仅引用不得重复定义
- **精简 SKILL.md**：移除用户营销内容，只保留触发条件、路由表、加载文件、全局边界
- **重构完整检测**：改为"先结论，后证据，最后只给必要修改"结构
  - 默认输出：总体结论 → 命中风险 → 修改建议
  - 可选增强：标题/标签/封面/评论区建议仅在用户回复"继续优化"后生成
- **降低视觉模板脆弱性**：
  - 移除 ASCII/Unicode 卡片框强制规范
  - 改为 Markdown 表格优先
  - Emoji 仅用于风险等级标识
  - 长文本片段控制在 30 字以内

## [1.3.1] - 2026-05-24

### Added
- 极客向重塑：Demo 案例替换为技术营销场景（独立开发/Vibe Coding/效率工具）
- risk-rubric 新增 5 项技术/商业类专属风险项：
  - 同质化技术黑话（+1）：底层逻辑、赋能、闭环、降维打击等
  - 过度硬核/机器感（+1）：直接复制 API 文档、纯参数罗列
  - 开源/工具导流诱导（+2）：Star 我的 GitHub、去官网下载
  - 技术效果夸大（+3）：O(1) 复杂度秒杀所有方案、100% 准确率
  - 伪技术概念包装（+2）：基于量子计算的推荐算法（实际未使用）

### Changed
- 优化 Demo 案例场景：从美妆种草转向技术营销（独立开发/Vibe Coding），更贴合 TRAE SOLO 开发者社区用户画像
