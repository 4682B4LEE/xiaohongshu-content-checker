## 使用流程

Skill 会根据用户输入自动识别意图，选择最合适的检测模式：

1. **意图识别**：分析用户输入，匹配唤醒词或 CLI 指令
2. **模式路由**：自动选择极速排雷 / 完整检测 / 批量检测 / 视频专项
3. **风险检测**：加载对应知识库，执行关键词匹配 + 语义分析（双引擎）
4. **输出报告**：先结论后证据，只列命中项，只改高风险和中风险

## 四大检测模式

| 模式 | 触发方式 | 适合场景 | 输出特点 |
|------|---------|---------|---------|
| **极速排雷** | `/fast <文本>` 或 "快速排雷：..." | 临时检测、时间紧迫 | 极简输出，只给结论和命中项 |
| **完整检测** | 直接粘贴笔记内容 或 `/full <文本>` | 正式发布前参考检查 | 先结论后证据，可选增强 |
| **批量检测** | `/batch` 或多条笔记 | 矩阵账号、内容团队 | 汇总报告 + 共性问题分析 |
| **视频专项** | `/video <文本>` | 视频创作者 | 叠加模式：基础检测 + 口播/字幕/画面风险 |

### CLI 快捷指令

## 设计规范

- **评分唯一来源**：所有模式共用 `risk-rubric.md`，避免规则碎片化
- **双引擎检测**：关键词匹配 + 语义分析，互补验证
- **先结论后证据**：用户第一眼看到风险等级和核心问题
- **只列命中项**：没有风险的项目不输出，避免信息过载
- **可选增强**：标题/标签/封面/评论区建议仅在用户回复"继续优化"后生成
- **不绝对判罚**：使用"存在推荐受限风险""可能触发审核"等建议性表述
- **防御性设计**：Anti-Jailbreak 策略，拒绝恶意探测、变体绕过、反向诱导
