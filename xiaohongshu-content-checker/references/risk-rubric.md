# 风险评分规则

> 本文件是 Skill 内唯一的风险评分来源。所有检测模式（fast/full/batch/video）在执行评分时，必须加载并遵循此文件。其他文件不得重复定义评分规则。

## 评分维度

### 通用风险项

| 风险项 | 分值 | 说明 |
|---|---:|---|
| 强导流 | +4 | 微信、VX、加我、私信购买、外部平台跳转等 |
| 医疗/金融/教育效果承诺 | +4 | 治愈、稳赚、保证就业、保证提分等 |
| 虚假宣传或绝对承诺 | +3 | 100%有效、永久、无效退款、保证结果等 |
| 广告法极限词 | +2 | 最好、第一、唯一、顶级、全网最低等 |
| 诱导互动 | +2 | 求赞、互粉、评论1发你、收藏过多少出教程等 |
| 标题党/焦虑制造 | +1 | 不看后悔、99%不知道、丑女逆袭等 |
| 同质化营销黑话 | +1 | 绝绝子、闭眼入、按头安利等 |

### 技术/商业类专属风险项

| 风险项 | 分值 | 说明 | 典型示例 |
|---|---:|---|---|
| 同质化技术黑话 | +1 | 过度堆砌技术/商业术语，可能导致营销浓度过高、自然流表现差 | "底层逻辑""赋能""闭环""降维打击""抓手""颗粒度""对齐""组合拳" |
| 过度硬核/机器感 | +1 | 文案过于像 GitHub Readme 或技术白皮书，缺乏情绪价值和场景代入 | 直接复制 API 接口文档、纯参数罗列、无个人体验描述 |
| 开源/工具导流诱导 | +2 | 以"开源""免费"为诱饵引导至外部平台 | "Star 我的 GitHub""去官网下载""加群获取安装包" |
| 技术效果夸大 | +3 | 对技术能力或性能做绝对化承诺 | "O(1) 复杂度秒杀所有方案""100% 准确率""零延迟" |
| 伪技术概念包装 | +2 | 用虚假或误导性技术概念包装普通功能 | "基于量子计算的推荐算法""AI 大模型驱动的自动排版"（实际未使用） |

## 重复计分规则

- 同一风险项重复出现时，最多计 2 次。
- 同一句中多个同类变体，只按最高风险项计一次。
- 强导流、医疗金融教育承诺不因上下文缓和而降为安全，只能通过删除或改写降低风险。

## 风险等级

| 命中项分值 | 总分 | 等级 | 发布建议 |
|:---:|---:|---|---|
| 单项 +1~+4 | 0 | 安全 | 可发布，仍建议人工复核 |
| 单项 +1~+4 | 1-3 | 低风险 | 建议微调 |
| 单项 +1~+4 | 4-6 | 中风险 | 建议修改后发布 |
| 单项 +1~+4 | 7+ | 高风险 | 建议重写高风险片段后再发布 |

## 表述边界

- 不使用"必限流""必封号""一定违规"。
- 使用"存在推荐受限风险""可能触发审核""建议发布前人工复核"。
- 如果依据来自经验总结而非官方规则，标注"基于经验判断"。

## 免责声明

平台规则会动态变化。本 Skill 基于公开规则、常见审核经验和内容运营实践提供风险提示，不代表平台官方判定。重要商业内容发布前建议人工复核。
