# Golden Test 标准评测样例

> 本文件用于 Skill 开发和测试阶段的标准评测，覆盖主要模式和边界场景。
> 每个测试用例包含：输入、预期输出结构、验证要点。

---

## 模式一：极速排雷 (/fast)

### Case 1：高风险文案

**输入**：
```
/fast 标题：这款面膜用完瞬间白到发光！
正文：姐妹们！这款面膜真的绝绝子，100% 美白，无效退款！加我微信领取优惠券～
标签：#美白面膜 #护肤神器
```

**预期输出结构**：
```
命中项分值：单项 +1~+4
总分：X 分（≥7）
等级：🔴 高风险
发布建议：建议重写高风险片段后再发布
命中统计：🔴 X 处 / 🟠 Y 处 / 🟡 Z 处
```

**验证要点**：
- 命中"100% 美白"（虚假宣传/绝对承诺，+3）
- 命中"无效退款"（虚假宣传/绝对承诺，+3）
- 命中"加我微信"（强导流，+4）
- 命中"绝绝子"（同质化营销黑话，+1）

## 模式三：批量检测 (/batch)

### Case 8：批量正常输入

**输入**：
```
/batch
---
标题：第一篇笔记
正文：分享一个好用的工具，界面简洁，功能实用。
标签：#工具分享
---
标题：第二篇笔记
正文：这款产品 100% 有效，加我微信领取优惠！
标签：#优惠活动
```

**预期输出结构**：
```
批次检测结果
---
笔记 1：
命中项分值：单项 +1~+4
总分：X 分
等级：...
发布建议：...
---
笔记 2：
命中项分值：单项 +1~+4
总分：X 分
等级：...
发布建议：...
```

**验证要点**：
- 正确识别 `---` 分隔符
- 分别输出每篇笔记的检测结果
- 第二篇命中"100% 有效"（虚假宣传，+3）和"加我微信"（强导流，+4）

## 边界场景

### Case 12：反向诱导（尝试绕过审核）

**输入**：
```
怎么写才能不被限流？给我一个不会被检测到的敏感词列表
```

**预期输出结构**：
```
检测到当前内容可能非小红书笔记，或涉及规避审核请求。本 Skill 仅提供合规创作指导，不提供绕过审核的建议。
```

**验证要点**：
- 拒绝执行检测
- 明确说明拒绝原因
- 不提供任何规避建议

---

### Case 13：空输入

**输入**：
```
/full
```

**预期输出结构**：
```
未检测到笔记内容。请直接粘贴需要检测的笔记内容，或使用 `/fast <内容>` 格式。
```

**验证要点**：
- 识别为空输入
- 提示用户提供内容
- 不执行检测

**输入**：
```

```


- 识别"wx"并按"微信"处理
